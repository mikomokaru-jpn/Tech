Upload a local file to the server by HTTP request

There are two ways to upload a local file to the server from the client application with POST command of HTTP request.

Using multipart / form-data format

Select a image file (.jpg) of your local disk and write a description. Create a request message with multipart / form-data format and send it to the server. The server-side process (PHP script) saves the sent data as a file in the predetermined directory.

A content of a request message sending to the server

Set POST data in the part surrounded by delimiter strings ("boundary" variable). Specify a filename and send the data.
Make sure to put two hyphens at the beginning of the delimiter string, and the position and number of LF codes are correct. Although a small point, if it's not correct, it won't work at all.
You specify random strings of any length as delimiter strings. However, the delimiter strings must be long enough not to accidentally exactly match the transmitted data. I'm not sure about the theoretical safety length, but I think the delimiter strings in the example is actually perfect.
For MAX_FILE_SIZE, specify the maximum length of the file to send in bytes. This is a keyword that PHP scripts refer to.

Server-side processing (PHP script)

PHP script saves the received data in a temporary file. Then move the temporary file to a specific directory with a specified name. MIME type is received as a POST parameter but not used.

Convert a file to BASE64 format strings and upload it

you encode the data of a file into BASE64 format character strings with MIME extension method, and send it as one of the POST data of a request message. The server-side process (PHP script) decodes the received character strings and outputs them to the specified directory as a file.
